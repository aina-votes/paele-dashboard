# Vote Paele — Voter Contact Progress Dashboard

Live dashboard showing doors knocked and phone calls made for Vote Paele's
non-GOTV phases, as a % of weekly / monthly / total goals. Refreshes every 30 minutes.

## Files

- `index.html` — single-page dashboard, no build step. Reads `progress.json`.
- `progress.json` — current counts + goals. Regenerated by `fetch_progress.py`.
- `fetch_progress.py` — pulls live counts from Solidarity Tech and writes `progress.json`.
- `screenshot.py` — local Playwright helper for visual QA (writes `screenshot_*.png`).

GOTV phases (`PAELE-SUPER-VTR (GOTV)` and `LĀHUI-REGULAR-PPL-(GOTV)`) are
deliberately excluded for now; add them to `PHASES` in `fetch_progress.py`
when the campaign moves into GOTV.

## Live data signals

- **Doors** — count of users in Vote Paele Voters (chapter 1790) whose
  `custom_user_properties['last-canvass-date']` is on/after the phase start.
- **Phones** — count of rows from `/calls` with `chapter_id = 1514` and
  `created_at` on/after the phase start.

`fetch_progress.py` uses the local `st_data/users.json` snapshot as a base and
overlays fresh `/users?_since=...` updates for recent canvass changes.

## Local run

```bash
python fetch_progress.py
python -m http.server 8000  # then open http://localhost:8000/
```

## Deploy to droplet (recommended)

The dashboard is a static page that reads a JSON file; the only thing that needs
the ST API key is `fetch_progress.py`. So:

1. Push this folder to `/root/fireflys-path/deployments/paele-dashboard/` on the
   droplet (already in repo via git pull pattern — see `reference_droplet.md`).
2. Serve `index.html` and `progress.json` via the existing nginx on the droplet
   under a subdomain like `paele-progress.ainavotes.com` (add Porkbun CNAME →
   droplet IP, then nginx server block pointing at this folder).
3. Add a cron entry:
   ```
   */30 * * * * cd /root/fireflys-path/deployments/paele-dashboard && /usr/bin/python3 fetch_progress.py >> /root/logs/paele-dashboard.log 2>&1
   ```

## Visual QA

```bash
python screenshot.py
```

Writes `screenshot_real.png` (live data) and `screenshot_mock_{total,week,month}.png`
(seeded mock numbers so the layout is visually verifiable even when counts are 0).
